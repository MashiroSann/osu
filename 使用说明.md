# OsuUiKit 使用说明

本文档说明如何在你的 .NET 项目中导入并使用本仓库中的 `OsuUiKit`。

## 1. 环境要求

- .NET SDK 8.0 或更高版本
- 可访问 NuGet 源（默认 `https://api.nuget.org/v3/index.json`）

## 2. 获取源码

将本仓库拉取到本地后，`uikit` 主项目路径为：

- `src/OsuUiKit/OsuUiKit.csproj`

## 3. 在你的项目中导入 OsuUiKit

在你的应用项目 `.csproj` 中添加项目引用：

```xml
<ItemGroup>
  <ProjectReference Include="../src/OsuUiKit/OsuUiKit.csproj" />
</ItemGroup>
```

> 说明：`Include` 的相对路径请按你的实际目录结构调整。

## 4. 还原并编译

在仓库根目录执行：

```bash
dotnet restore
```

单独编译 uikit：

```bash
dotnet build src/OsuUiKit/OsuUiKit.csproj -v minimal
```

## 5. 代码中引入命名空间

按需在 C# 文件顶部引入：

```csharp
using OsuUiKit;
using OsuUiKit.Containers;
using OsuUiKit.Containers.Markdown;
using OsuUiKit.Sprites;
using OsuUiKit.UserInterface;
using OsuUiKit.UserInterface.PageSelector;
using OsuUiKit.UserInterfaceV2;
```

## 6. 调用控件

控件完整列表、对应调用方式、所需命名空间见根目录文档：

- `List.md`

## 7. 常见问题排查

1. **编译报依赖错误**
   - 确认已执行 `dotnet restore`
   - 确认网络可访问 NuGet 源
   - 确认 SDK 版本为 .NET 8+

2. **命名空间找不到**
   - 确认已添加 `ProjectReference`
   - 确认顶部 `using` 已按控件所在命名空间引入

3. **控件无法实例化**
   - 部分控件为泛型控件，需要补充类型参数后再实例化
