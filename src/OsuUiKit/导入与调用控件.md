# OsuUiKit 导入与调用控件指南

本文档说明如何把 `OsuUiKit` 导入到你的项目，并开始调用控件。

## 1. 环境要求

- .NET SDK 8.0+
- 可访问 NuGet 官方源（`https://api.nuget.org/v3/index.json`）

## 2. 导入方式

推荐使用“项目引用”方式。

### 方式 A：同仓库直接引用

在你的应用项目 `.csproj` 中添加：

```xml
<ItemGroup>
  <ProjectReference Include="..\src\OsuUiKit\OsuUiKit.csproj" />
</ItemGroup>
```

### 方式 B：复制目录后引用

将 `src/OsuUiKit` 复制到你的仓库后，再按相对路径添加 `ProjectReference`。

## 3. 依赖说明

`OsuUiKit` 已在 `OsuUiKit.csproj` 中声明所需依赖：

- `ppy.osu.Framework`
- `ppy.osu.Game`
- `ppy.osu.Game.Resources`

你通常不需要在业务工程重复声明这些包（除非你要强制锁定版本）。

## 4. 还原与构建

```bash
dotnet restore
dotnet build
```

若构建失败，请优先检查：

1. NuGet 源是否可访问；
2. 你的项目是否使用 .NET 8.0 或更高版本；
3. 是否已经正确添加 `ProjectReference`。

## 5. 调用控件的基本步骤

1. 在代码中引入命名空间：
   - `OsuUiKit`
   - `OsuUiKit.UserInterface`
   - `OsuUiKit.UserInterfaceV2`
2. 在你的 `Drawable` / `CompositeDrawable` / `Game` 中实例化控件。
3. 将控件添加到容器（如 `FillFlowContainer`、`Container`）进行布局。
4. 按需绑定事件、可绑定值（`Bindable<T>`）或配置项。

## 6. 建议

- 先运行 `OsuUiKit.Demo` 验证环境，再集成到业务项目。
- 新增控件时尽量保持只依赖 `OsuUiKit` 已声明的包，避免再次耦合到不必要模块。
